// detection_stack.md

// METHODOLOGY

Six detection methods, each with rules anyone can audit and replicate. Source code MIT-licensed on GitHub.

#01

FUNDING_SOURCE_CLUSTERING

clustering/funding_cluster.py
  • › Group addresses by their first incoming non-CEX funder
  • › Filter out groups with <3 members
  • › Confidence scales by time spread: <24h → 0.95, <7d → 0.80, else 0.60
  • › CEX hot wallets (Binance, Coinbase, Huobi) explicitly excluded
#02

BEHAVIORAL_CLUSTERING

clustering/behavior_cluster.py
  • › Build 10-feature vector per address (timing, entropy, protocol diversity, etc.)
  • › Drop zero-variance columns (σ < 1e-12) to prevent uniform-input failure
  • › z-score normalize → HDBSCAN with min_cluster_size=5, leaf cluster selection
  • › Fallback: if all-noise AND ≥3 features zero-variance, treat batch as one Sybil cohort
  • › Intra-cluster spread → confidence (lower = higher)
#03

GRAPH_COMMUNITY_DETECTION

clustering/graph_community.py
  • › Build directed weighted graph from in-batch transactions (igraph, 500K+ nodes)
  • › Leiden algorithm via leidenalg (RBConfigurationVertexPartition, resolution=1.0, seed=42)
  • › Filter communities by size (5 ≤ n ≤ 500) and density (≥ 0.3)
  • › Density threshold cuts legitimate DEX-using groups while keeping farms
#04

CROSS_CHAIN_IDENTITY_LINKING

clustering/cross_chain.py
  • › Deterministic: bridge tx A→B across chains = same entity (Stargate, Hop, Across, Wormhole)
  • › Probabilistic: same funder + same timestamp window (±10min) across chains
  • › Union-find aggregates into entity components
  • › Cluster fires when ≥3 chain-nodes belong to one entity
#05

TEMPORAL_ANOMALY_FEATURES

features/temporal.py
  • › Hour-of-day entropy (humans > 2.5, bots < 1.5)
  • › Day-of-week entropy
  • › Min inter-tx seconds (humans rarely < 60s)
  • › Burst score (% of txs in 10% of activity window)
  • › Lag-1 autocorrelation of inter-tx gaps
  • › Activity ratio (active days / account age)
#06

ML_ENSEMBLE_SCORING

scoring/train.py
  • › LightGBM binary classifier with class_weight='balanced'
  • › Sample-weighted by confidence tier (T1=4.0, T4=0.8, G1=4.0, G2=2.0)
  • › Trained on ALL tiers (T1-T5 + G1-G2 with weights)
  • › Evaluated on T1+T2+G1 holdout ONLY — never against T4 outputs
  • › Honest metrics: P@70, R@70, FPR-on-G1, ROC-AUC, adversarial recall

// label_tier_system

We don't pretend our labels are perfect. Each labelled address has a tier.

// what_we_DONT_do

  • › Score based on token holdings or wealth
  • › Penalise mixer / privacy-tool usage per se
  • › Flag addresses just because they hold a Worldcoin orb or any other "humanness" credential
  • › Publicly list any individual address as Sybil — only aggregates
  • › Train on labels from other commercial detectors
  • › Charge for the appeal process

// reproducibility

Every model artifact stores its feature_schema_hash and training_manifest_hash. If you can't reproduce a score from the published artifact + manifest hashes, the score is invalid.
