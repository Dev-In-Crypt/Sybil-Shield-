// how_it_works

Submit an address list. We ingest on-chain history, run six independent detection methods, produce a per-address risk score with evidence, and return both a queryable result set and a downloadable CSV. The whole pipeline takes hours, not weeks.

1

REGISTER

Create a free account, receive an API key shown once. Free tier covers 100 calls/month.

2

CREATE_ANALYSIS

Submit a list of wallet addresses (up to 1M per request) with chains and sensitivity. The platform queues the job and returns an analysis ID.

3

TRACK_PROGRESS

Poll the analysis endpoint or subscribe to a webhook. Status transitions: ingesting → analyzing → scoring → complete.

4

READ_RESULTS

Fetch scored addresses (filterable by label), detected clusters, and per-address evidence. Export as CSV.

5

HANDLE_APPEALS

Flagged users submit appeals directly via the public endpoint. Every submission is logged immutably and routed to a reviewer.

// what_you_get_back

SYBIL_SCORE

0-100 confidence the address is part of a Sybil farm. Three bucket labels: genuine, suspicious, sybil.

CLUSTER_ID

Which other wallets in the analysis it groups with, plus cluster size and detection method.

EVIDENCE_LIST

Per-address human-readable evidence items pointing at specific features or cluster memberships.

CONFIDENCE

Calibrated probability output from the ML ensemble alongside per-evidence-item confidence.

CSV_EXPORT

Full results table for compliance archiving or distribution-list filtering.

WEBHOOK_NOTIFICATION

Signed POST to your URL on analysis completion with summary statistics.

  • webhook_subscribeSet URL, receive signing secret

  • create_analysisSubmit address list, chains, sensitivity
  • list_analysesPaginated history
  • get_statusIncludes summary when complete
  • get_resultsPaginated, filter by label
  • export_csvStream the full results table
  • get_clustersCluster metadata with detection method

/scoring

  • single_scoreCached lookup for any address previously seen
  • batch_scoreBulk lookup endpoint (up to 100)
  • entity_checkCross-analysis intelligence — flagged anywhere?

/trust

  • feedbackCustomer-side FP/FN reporting
  • public_appealOpen, unauthenticated — dispute your score
  • appeals_policyPublic JSON describing review SLA

// webhook_signature

Every webhook payload is signed with HMAC-SHA256 using a secret you receive when you configure the webhook URL. The signature arrives in x-sybilshield-signature as sha256=<hex>. Recompute the HMAC over the raw request body and compare with constant-time equality. Reject mismatches.
